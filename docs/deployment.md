# Kursu Deployment Plan

## Current Target

Kursu deploys as a single Netlify site:

- Frontend: Vite build output in `dist/`
- API: Netlify Functions from `netlify/functions`
- Async jobs: Netlify Background Functions generated by `npm run sync:bg`
- Database: Netlify Database / Postgres with `pgvector`
- Secrets: Netlify environment variables plus local `.env.local`

## Pre-Deploy Checklist

1. Rotate any API keys that were shared outside the secret manager.
2. Use Node 20.12.2+ locally and Netlify CLI 26+ (`npx netlify --version`).
3. Set Netlify environment variables:
   - `ANTHROPIC_API_KEY`
   - `VOYAGE_API_KEY`
   - `KURSU_DEMO_TEACHER_ID=teacher_demo_001`
   - `KURSU_LOG_LEVEL=info`
4. Create or connect Netlify Database for the site.
5. Confirm the deployed runtime exposes the database URL expected by `db/client.ts`.
6. Run migrations once against the production database:
   - `npm run db:migrate`
7. Optionally seed demo data:
   - `npm run db:seed`
8. Build locally:
   - `npm run typecheck`
   - `npm run smoke:chunking`
   - `npm run build`

## Background Functions

Canonical source files live in `netlify/background-functions` as P3-owned files. Before dev/build, `npm run sync:bg` copies them into `netlify/functions` with Netlify-compatible names:

- `background-embed-notes.ts` -> `embed-notes-background.ts`
- `background-embed-transcript.ts` -> `embed-transcript-background.ts`
- `background-generate-feedback.ts` -> `generate-feedback-background.ts`

The `-background` suffix is required for Netlify to deploy them as asynchronous background jobs rather than normal synchronous functions.

## Non-Negotiable Architecture Rule

Transcript ingestion is not an LLM call.

During a live lecture, `/api/lectures/:id/transcript/append` only validates and stores transcript segments. It may trigger `embed-transcript-background`, but that background job only creates Voyage embeddings. It must not call Claude, Sonnet, Haiku, Answer, Router, Flagger, or Feedback.

LLM calls are allowed only:

- when a student asks a question through `question-ask.ts`
- when a teacher ends a lecture and `generate-feedback-background` runs once

This keeps live ingestion cheap, private, and scalable.

## Netlify Rollout Steps

1. `netlify init` and link the repo to a new Netlify site.
2. Configure build command `npm run build`, publish directory `dist`, functions directory `netlify/functions`.
3. Create or connect Netlify Database.
4. Add environment variables in Netlify UI or CLI.
5. Run `npm run db:migrate` with the production database URL available.
6. Push to `main` and verify the deploy log includes three background functions:
   - `embed-notes-background`
   - `embed-transcript-background`
   - `generate-feedback-background`
7. Smoke deployed endpoints:
   - `GET /api/health`
   - `POST /api/lectures`
   - `POST /api/lectures/:id/notes`
   - `POST /api/lectures/:id/start`
   - `POST /api/lectures/:id/questions`
   - `POST /api/lectures/:id/end`
   - `GET /api/lectures/:id/feedback`
